Vorläufiges Setup:

== Backend

Zum einfacheren Debuggen läuft der FastCGI-Prozess separat (wird also nicht vom
Webserver gestartet). Gestartet wird er folgendermaßen:

cd core/App-Emit
perl fastcgi.pl

== lighttpd

Folgendes zur Config hinzufügen:

fastcgi.server = (
	"/" => (
		(
			"host" => "127.0.0.1",
			"port" => 4441,
			"check-local" => "disable",
		),
	),
)

== testen

curl -X POST http://localhost:81/write '{"api": "yay"}'

== Frontend

Derzeit über normales CGI eingebunden (FastCGI kommt noch), in einem Apache:

<VirtualHost *:9070>
	<Directory "/home/michael/emit/frontends/web">
		Options Indexes FollowSymLinks +ExecCGI
		AllowOverride None
		Order allow,deny
		Allow from all
	</Directory>

	DocumentRoot /home/michael/emit/frontends/web

	AddHandler cgi-script cgi pl
</VirtualHost>

Die URL für einen neuen Bugreport lautet dann:

http://localhost:9070/App-emit-Web/cgi.pl/new

Derzeit sind die Pfade hard-coded.
